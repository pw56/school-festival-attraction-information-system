# 待ち時間のデータ送信のクライアント(現場デバイスで稼働するウェブアプリ)

## 備考
- QRコードで、出し物のUUIDを読み込む

## 処理
1. デバイスの負荷を抑えるため、1秒に1回カメラ映像からグループの数をカウント
  1. MediaPipe Object Detecterで人物を検出して、ROI抽出
  2. MediaPipe Face Landmarkerを用いてそれぞれの人の顔の向きを検出してグループ候補を検出
  3. 前後のフレームの結果を組み合わせてグループの確定
2. 送信の頻度が比較的少ないので、10秒に1回、Fetch APIでクラウドに待機列の人数とグループの数を送信
3. 無理のない範囲でメモリを定期的に解放するため、10分に1回ビデオストリームを廃棄して再生成
