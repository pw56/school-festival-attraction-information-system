# 機能

## 出し物の個別ページ
- サイトには出し物のサムネイルを設定できる
- サイトには出し物の概要を掲載できる
- サイトには某テーマパークのアプリのように出し物の特徴や注意事項を書ける

例
```
profiles
└─ exciting_coaster
   ├─ thumbnail.png
   └─ setting.yml
```

## 全体
- 出し物が複数登録されている場合は、某アプリのように横にスライドして見れるようにする
- 個別ページはクラウドから出し物一覧のJSONを取得して、出し物が複数あるか確認する
- 出し物複数ある場合は、横にスライドした時に某アプリのように別の出し物のページに切り替わるようにする
- これらの仕様から、一つのページで個別ページはクエリ文字列で区別し、個別の出し物の情報は純粋にサムネイルとプロフィールのJSONを取得してクライアントで再構成する仕様にする
- クエリ文字列とHistory APIを使用する

## 備考
- ユーザーエージェントや案内のポップアップ組み合わせれば、LIFF使わずに整理券配布も無理ではない
- 当然だが、同じ時間帯に重複して整理券取得は禁止する
- 整理券に時間指定機能もつける
- 整理券にキャンセル機能もつける
- 時間指定の整理券に、キャンセルが出たら予約機能もつける
