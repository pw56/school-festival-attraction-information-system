# アーキテクチャーのメモ

## 待ち時間のデータ送信のクライアント(現場デバイスのウェブページ)
1. デバイスの負荷を抑えるため、1秒に1回カメラ映像からペアの数をカウント
  1. MediaPipe Object Detecterで人物を検出して、ROI抽出
  2. MediaPipe Face Landmarkerを用いてそれぞれの人の顔の向きを検出してペア候補を検出
  3. 前後のフレームの結果を組み合わせてペアの確定
2. 送信の頻度が比較的少ないので、10秒に1回、Fetch APIでクラウドに待機列の人数とペアの数を送信
3. 無理のない範囲でメモリを定期的に解放するため、10分に1回ビデオストリームを廃棄して再生成

## クラウドのバックエンド

### 出し物情報取得APIサービス
1. 全ての出し物の設定ファイルから情報を抽出して、1つのJSONファイルに結合
2. JSONファイルとして配信
※ サーバー側の_headersファイルで、ブラウザのJSONのファイルのキャッシュを阻止し、APIとして振る舞う

### 待ち時間サービス
1. サーバーレス環境でクライアントからリクエストを受け付け
2. 現場のデバイスから送られてきた情報をデータベースに記録
3. 設定に応じて待ち時間を計算
  - 設定ファイルによる固定式の計算の設定の場合
    1. サーバーレス環境で、設定ファイルで設定された1組あたりの待ち時間から待ち時間を計算する
  - リアルタイムのデータに応じた動的な計算の設定の場合
    1. データベースから直近の推移を取得して、今後の1組あたりの待ち時間を予測
    2. 現在の待機列のペアの数から待ち時間を計算
    3. 混雑状況サービスにも登録されていれば連携
4. 待ち時間の計算結果をデータベースに保存

### 混雑状況サービス
1. サーバーレス環境でクライアントからリクエストを受け付け
2. 現場のデバイスから送られてきた情報をデータベースに記録
3. 今後の混雑状況を予測
4. 混雑状況の予測結果をデータベースに保存

### リアルタイムデータ取得APIサービス
1. 出し物の待ち時間・混雑などのデータベースの情報を1つのJSONファイルに結合
2. JSONファイルとして配信
※ サーバー側の_headersファイルで、ブラウザのJSONのファイルのキャッシュを阻止し、APIとして振る舞う

## 出し物情報掲載ページ
- クラウドでウェブページをホスティング
- 整理券を実装する場合はPWA
- ウェブページは初回に、出し物情報APIで出し物のプロフィールを取得する
- 定期的にリアルタイムデータ取得APIで、出し物のリアルタイムの情報(待ち時間や混雑状況など)を取得する
