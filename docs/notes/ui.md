# UIのメモ

## モバイル版
某テーマパークの本家アプリのように
1. サムネイル
2. 出し物名
3. 待ち時間
4. マップ
5. その他情報
のように表示する。
大まかな要素は次の通りである。
- サムネイル: 円形の画像、最も大きい要素
- 出し物名: H1
- 待ち時間: H2
- マップ: アイコンとラベルのボタン、タップすると画面が暗転し、ポップアップで表示、画像の右上に閉じるボタンも
- その他情報: ラベルがH2で、その下にコンテンツ(テキスト)が表示される
セマンティックHTMLを意識してする。
このシステムは、複数の出し物のページを同じシステム上で運用するケースに対応しているため、出し物ごとのページでは、要素はページ直下ではなく丸みを帯びたカード上に配置される。
カードは真っ白で、背景の親(ページ)は少し暗い白である。
カードの横幅はあえてページの横サイドが少し余るように作られている。
同じシステム上に複数の出し物が登録されている場合は、その横サイドの余白に別の出し物のカードがチラ見えするようになっていて、同時に影も出すことで横方向のスクロールを促している。
横方向のスクロールはカードが中央に来たらピタッ！と止まるよくある方式。
横サイドのカードは中央のカードよりも小さく暗く、中央にスライドされた時にアニメーションで切り替わる。

## デスクトップ版
基本的にモバイル版と同じ。
サムネイルと待ち時間とマップボタンを左側、その他の情報を右側にして、2カラムレイアウトにする。
